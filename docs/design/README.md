# Magic Shuffle — design & engine notes

Historical engineering notes from the first static prototype. The shipping app lives under **`public/`** (UI) and **`src/`** (API + recommendation). Use these documents for **visual language**, **component ideas**, and **deterministic scoring** reference — file names in the “prototype” column are legacy.

## Files

| `02-component-spec.md` | Component map (conceptual; original used inline JSX) |

## What’s mocked vs. real (current build)

- **Real now:** **Weather** (Open-Meteo, keyless, IP-located), **Calendar** (Google
  Calendar OAuth, read-only, with mock fallback), OpenAI planning, Spotify metadata for
  proposed tracks, optional user OAuth for Web Playback.
- **Still mocked:** **Biometrics** — generated by `src/lib/generateBiometrics.js`
  (research-based, internally consistent). Google Health / Zepp are deliberately OFF.
- **Output:** a full arc-ordered **playlist** (ISO-principle over the circumplex model),
  not a single pick. Engine: `src/lib/affect.js` + `src/lib/isoPlaylist.js`. The legacy
  deterministic scorer (`score.js`) is kept for the old single-pick path only.

> The `03-engine-spec.md` weighted-scorer doc below is the **legacy** single-pick spec.
> The shipping playlist path uses the affect/arc engine instead.
